@sindresorhus/slugify is a small ESM-only Node.js library (≥ Node 20) that turns arbitrary strings into URL/filename/ID-safe slugs. The implementation lives almost entirely in index.js (~140 lines), with a tiny data file for default symbol replacements, a TypeScript declaration file, and an Ava-based test suite. The main entry exports a default `slugify(string, options?)` plus a stateful `slugifyWithCounter()` that appends `-2`, `-3`, ... to repeated inputs.

## Behavioral subtleties and gotchas

- **`transliterate: false` silently disables built-in replacements like `&` → `and`** _(medium)_
  When `transliterate` is false the code skips the merge with `builtinOverridableReplacements` and only applies the user's `customReplacements`. A test asserts that `'foo & bar'` becomes `'foo-bar'` (not `'foo-and-bar'`) in that mode — a subtle behavioral cliff for callers who toggle transliteration off for performance.
  - [`index.js:66-78`](https://github.com/sindresorhus/slugify/blob/7c318bd1aa4b4affab29761f15a9604323fe2a3b/index.js#L66-L78)
  - [`test.js:247-249`](https://github.com/sindresorhus/slugify/blob/7c318bd1aa4b4affab29761f15a9604323fe2a3b/test.js#L247-L249)
- **`preserveCharacters` overlapping `separator` throws** _(low)_
  `buildPatternSlug` throws an `Error` if any preserved character equals the separator. Callers that build options dynamically must guard against this; tests confirm two throwing cases.
  - [`test.js:211-216`](https://github.com/sindresorhus/slugify/blob/7c318bd1aa4b4affab29761f15a9604323fe2a3b/test.js#L211-L216)
- **`slugify` calls `string.replace(patternSlug, ...)` instead of `replaceAll`** _(low)_
  The disallowed-character regex is created with the `g` flag (or `gu`) so a single `.replace` does replace all matches — but the asymmetry vs. nearby `.replaceAll` calls is easy to misread when editing. Any future change that drops the `g` flag from `buildPatternSlug` would silently break slug generation.
  - [`index.js:42-43`](https://github.com/sindresorhus/slugify/blob/7c318bd1aa4b4affab29761f15a9604323fe2a3b/index.js#L42-L43)
  - [`index.js:94-95`](https://github.com/sindresorhus/slugify/blob/7c318bd1aa4b4affab29761f15a9604323fe2a3b/index.js#L94-L95)
- **`slugifyWithCounter` state is per-instance and never auto-resets** _(medium)_
  Each call to `slugifyWithCounter()` returns its own closure over a `Map`. Sharing one instance across unrelated documents/requests will cause counts to bleed across them; callers must call `.reset()` themselves. Test 'counter' demonstrates how aggressively state accumulates.
  - [`test.js:172-196`](https://github.com/sindresorhus/slugify/blob/7c318bd1aa4b4affab29761f15a9604323fe2a3b/test.js#L172-L196)
- **Possessive stripping only handles `'s` and `'t` followed by whitespace or end-of-string** _(low)_
  The regex requires the suffix character to be exactly `t` or `s` and then `(\s|$)`. Possessives mid-word like `Conway's-Law` (apostrophe followed by `-`) would not be cleaned up the same way, since the lookahead doesn't include the separator. Worth knowing when debugging unexpected hyphenation around quoted text.
  - [`index.js:90-92`](https://github.com/sindresorhus/slugify/blob/7c318bd1aa4b4affab29761f15a9604323fe2a3b/index.js#L90-L92)
