`p-limit` is a tiny, ESM-only utility library that caps how many promise-returning functions run concurrently. The entire implementation is 128 lines in `index.js`, backed by a `yocto-queue` linked-list. It exposes a default `pLimit(concurrency)` factory returning a callable `LimitFunction` (with `activeCount`, `pendingCount`, `concurrency`, `clearQueue`, `map`) and a named `limitFunction(fn, options)` helper that wraps a single function. Requires Node.js ≥ 20; tested via `xo`, `ava`, and `tsd` on Node 20/22/24.

## How the limiter is implemented

- **A `yocto-queue` instance holds pending items; `activeCount` tracks running tasks**
  `enqueue` pushes a `{reject, run}` item, `resumeNext` dequeues and invokes `run()` when `activeCount < concurrency`, and `next` decrements after each task and tries to resume.
  - [`index.js:16-30`](https://github.com/sindresorhus/p-limit/blob/42599ebbbb1228a5bdab381fcf8f4ac20eb8d551/index.js#L16-L30)
  - [`index.js:50-64`](https://github.com/sindresorhus/p-limit/blob/42599ebbbb1228a5bdab381fcf8f4ac20eb8d551/index.js#L50-L64)
- **Async-context preservation is intentional**
  Each queue item wraps its trigger in `new Promise(internalResolve => …)` and uses `.then(run.bind(undefined, …))` so the original async execution context (e.g., `AsyncLocalStorage`) is retained when the task eventually runs.
  - [`index.js:53-58`](https://github.com/sindresorhus/p-limit/blob/42599ebbbb1228a5bdab381fcf8f4ac20eb8d551/index.js#L53-L58)
  - [`test.js:45-58`](https://github.com/sindresorhus/p-limit/blob/42599ebbbb1228a5bdab381fcf8f4ac20eb8d551/test.js#L45-L58)
- **`run` forwards the result promise to the caller before awaiting it, and swallows the awaited error**
  `resolve(result)` is called immediately so the original caller sees both fulfillments and rejections; the local `await result` is wrapped in `try { … } catch {}` purely to avoid an unhandled-rejection warning before `next()` runs.
  - [`index.js:32-48`](https://github.com/sindresorhus/p-limit/blob/42599ebbbb1228a5bdab381fcf8f4ac20eb8d551/index.js#L32-L48)
- **Increasing `concurrency` drains pending tasks inside a `queueMicrotask`**
  The setter validates the new value and schedules a microtask that calls `resumeNext()` repeatedly while there is slack, avoiding reentrant synchronous draining.
  - [`index.js:91-104`](https://github.com/sindresorhus/p-limit/blob/42599ebbbb1228a5bdab381fcf8f4ac20eb8d551/index.js#L91-L104)
- **`clearQueue` either drops or rejects pending items**
  When `rejectOnClear` is false (default), it calls `queue.clear()`. When true, it dequeues every pending item and rejects it with `AbortSignal.abort().reason` (a DOMException named `AbortError`).
  - [`index.js:77-89`](https://github.com/sindresorhus/p-limit/blob/42599ebbbb1228a5bdab381fcf8f4ac20eb8d551/index.js#L77-L89)
  - [`test.js:199-215`](https://github.com/sindresorhus/p-limit/blob/42599ebbbb1228a5bdab381fcf8f4ac20eb8d551/test.js#L199-L215)
- **`limit.map` is a thin wrapper over `Promise.all`**
  It builds `Array.from(iterable, (value, index) => this(function_, value, index))` and awaits all of them — supports arrays, sets, and iterators.
  - [`index.js:106-111`](https://github.com/sindresorhus/p-limit/blob/42599ebbbb1228a5bdab381fcf8f4ac20eb8d551/index.js#L106-L111)

## Behavioral gotchas worth knowing

- **Reentrant calls into the same limiter can deadlock** _(medium)_
  The README explicitly warns: a function already running under a limiter that calls the same `limit` again can starve, because no slot frees up until the outer function returns. Use a separate limiter for inner tasks.
  - [`readme.md:76-76`](https://github.com/sindresorhus/p-limit/blob/42599ebbbb1228a5bdab381fcf8f4ac20eb8d551/readme.md#L76-L76)
  - [`index.d.ts:49-49`](https://github.com/sindresorhus/p-limit/blob/42599ebbbb1228a5bdab381fcf8f4ac20eb8d551/index.d.ts#L49-L49)
- **`clearQueue()` without `rejectOnClear` leaves dropped promises forever pending** _(medium)_
  By default, `queue.clear()` discards items without resolving or rejecting them; their externally-visible promises never settle, so `Promise.all` on them will hang. The README/recipes recommend `rejectOnClear: true` or `Promise.allSettled` for graceful shutdown.
  - [`index.js:77-89`](https://github.com/sindresorhus/p-limit/blob/42599ebbbb1228a5bdab381fcf8f4ac20eb8d551/index.js#L77-L89)
  - [`readme.md:96-105`](https://github.com/sindresorhus/p-limit/blob/42599ebbbb1228a5bdab381fcf8f4ac20eb8d551/readme.md#L96-L105)
  - [`recipes.md:76-114`](https://github.com/sindresorhus/p-limit/blob/42599ebbbb1228a5bdab381fcf8f4ac20eb8d551/recipes.md#L76-L114)
- **`clearQueue()` does not cancel already-running tasks** _(low)_
  Only pending entries in the queue are dropped/rejected; tasks already passed to `run()` continue until they settle.
  - [`readme.md:100-102`](https://github.com/sindresorhus/p-limit/blob/42599ebbbb1228a5bdab381fcf8f4ac20eb8d551/readme.md#L100-L102)
  - [`index.d.ts:22-22`](https://github.com/sindresorhus/p-limit/blob/42599ebbbb1228a5bdab381fcf8f4ac20eb8d551/index.d.ts#L22-L22)
- **ESM-only — no CommonJS build is published** _(low)_
  `package.json` sets `"type": "module"` and the `exports` map exposes only `./index.js`. Consumers on CommonJS toolchains need dynamic `import()` or interop.
  - [`package.json:13-17`](https://github.com/sindresorhus/p-limit/blob/42599ebbbb1228a5bdab381fcf8f4ac20eb8d551/package.json#L13-L17)
- **Node 20 test for AbortError-rejection is skipped due to a runtime bug** _(low)_
  `test.js` conditionally skips the `rejectOnClear` rejection test on Node 20 because of a DOMException interaction with AVA's `throwsAsync`. The feature itself ships, but isn't exercised by CI on Node 20.
  - [`test.js:196-215`](https://github.com/sindresorhus/p-limit/blob/42599ebbbb1228a5bdab381fcf8f4ac20eb8d551/test.js#L196-L215)
- **Errors awaited inside `run` are silently swallowed locally** _(info)_
  The `try { await result; } catch {}` in `run` is required to keep `next()` reachable and to avoid unhandled-rejection noise, but it means any logic added between the `await` and `next()` will not see the failure. The caller still sees the rejection via `resolve(result)`.
  - [`index.js:32-48`](https://github.com/sindresorhus/p-limit/blob/42599ebbbb1228a5bdab381fcf8f4ac20eb8d551/index.js#L32-L48)
